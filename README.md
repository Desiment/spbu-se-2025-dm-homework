# Репозиторий-шаблон для выполнения домашних заданий

Репозиторий-шаблон для ДЗ группы Б.73 СПбГУ 2025 года набора.

## Структура репозитория

```
.
├── prepare.sh         # Скрипт подготовки решений перед тестированием
├── solutions/         # Директория с решениями задач
│   ├── solution-01.py      # Файл решения задачи 1
│   └── ...                 # Файлы решений других задач
└── tests/             # Директория с тестами (заполнено за вас)
    ├── inputs/             # Входные данные для тестов
    ├── outputs/            # Ожидаемые выходные данные/проверка ответов
    └── runner/             # Запускатель тестов
```

## Процесс сдачи заданий

Ваш репозиторий создается как закрытый форк данного репозитория. Меня необходимо добавить как коллаборатора в форк.

1. Для каждой задачи создавайте отдельную ветку (для примера ветка solution-01 уже есть: её можно не форкать):
```bash
git checkout -b solution-01
```
2. Размещайте код своих решений в папке `solutions/`
    - Каждая программа должна:
        - Читать входные данные из stdin
        - Выводить результат в stdout в формате, указанном в условии задачи
    - Заполните скрипт ``prepare.sh``, который после запуска выдаёт исполняемый файл под каждую задачу. Сейчас есть пример заполнения под Python
3. Тестирование
    - Создайте Pull Request из вашей ветки в `master`
    - В названии PR укажите номер задачи (например: "Solution 01")
    - GitHub Workflow автоматически запустит тесты

## Локальное тестирование

Перед созданием PR рекомендуется протестировать решение локально:
```bash
# Подготовить решения
./prepare.sh

# Запустить тесты (если доступен тестовый скрипт)
./tests/runner/run.py
```
